# 퀴즈 데이터 스키마 가이드

새로운 주제(소수, 곱셈 등)를 추가할 때 사용하는 데이터 구조 정의서입니다.

## 1. 파일 구조
`/src/data/` 폴더 내에 주제별로 파일을 생성합니다. (예: `decimalsQuizzes.js`, `multiplicationQuizzes.js`)

## 2. 데이터 형식 (JSON/JS Object)
각 단원(unit)을 키로 가지며, 서술형 제목과 문제 리스트를 포함합니다.

```javascript
export const themeQuizzes = {
  'unit1': {
    title: '단원 제목 (예: 소수의 개념)',
    questions: [
      {
        id: 'theme-1-1', // 고유 ID (주제-단원-번호)
        question: '질문 텍스트 (수식은 $기호로 감쌉니다)',
        options: ['보기1', '보기2', '보기3', '보기4'],
        answer: '정답 텍스트 (옵션 중 하나와 일치해야 함)',
        hint: '오답 시 보여줄 힌트'
      },
      // ... 15문항 권장
    ]
  }
}
```

## 3. 작성 규칙
1. **수식 표현**: LaTeX 문법을 사용합니다. 수식은 `$ \frac{1}{2} $`와 같이 `$`로 감쌉니다.
2. **수식 내 한글**: 수식 기호(`$`) 안에는 한글을 직접 넣지 마세요. 
   - 잘못됨: `$분수$`
   - 올바름: `분수` 또는 `$ \text{분수} $`
3. **정답 일치**: `answer` 속성의 텍스트는 `options` 리스트에 포함된 문자열과 글자 하나 틀리지 않고 일치해야 합니다. (공백 주의)
4. **ID 고유성**: 중복되지 않는 ID를 부여하여 파이어베이스 이력 관리 시 혼선이 없도록 합니다.
